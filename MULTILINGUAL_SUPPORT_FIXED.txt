HOW IT WORKS:
═════════════════════════════════════════════════════════════════════════════

FLOW:
1. User first visits app (http://localhost:3000)
   ↓
2. If no language selected → Shows /language page
   ↓
3. User selects language from beautiful card interface
   ↓
4. Language saved to localStorage
   ↓
5. App redirects to /login with selected language active
   ↓
6. User can still change language in login page dropdown
   ↓
7. After login, selected language persists throughout app
   ↓
8. Language setting saved in MongoDB user profile (for future implementation)

USING TRANSLATIONS IN COMPONENTS:
═════════════════════════════════════════════════════════════════════════════

Example - Using translations in any component:

import { useTranslation } from 'react-i18next';

function MyComponent() {
  const { t, i18n } = useTranslation();
  
  return (
    <div>
      <h1>{t('common.welcome')}</h1>
      <button onClick={() => i18n.changeLanguage('hi')}>
        हिन्दी
      </button>
    </div>
  );
}

Available translation keys:
- nav.dashboard, nav.cropAnalysis, nav.soilHealth, etc.
- common.welcome, common.selectLanguage, common.settings, etc.
- auth.login, auth.register, auth.email, auth.password, etc.
- dashboard.overview, dashboard.statistics, etc.
- crops.cropAnalysis, crops.cropName, etc.
- soil.soilHealth, soil.nitrogen, etc.
- weather.weather, weather.temperature, etc.
- disease.diseaseDetection, disease.uploadImage, etc.
